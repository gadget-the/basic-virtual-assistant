i could combine the like and dislike categories into one "sentiment" category, then after classification generate a number representation for said sentiment
each verb(main sentiment word) would have a base value
    {
        "love": 2,
        "like": 1,
        "dislike": -1,
        "hate": -2
    }

    whereas each adverb(secondary sentiment word) would have a multiplier value
    {
        "really": 2,
        "don't": -1,
        "so much": 2(?)
    }
    (I'm not sure if "don't" would be a secondary sentiment word or a "tertiary sentiment word", either way it would flip the sign; -1)
    
    the overall sentiment value for a sentence would be the base value times the multipier value
        "I love cats" --> "love" --> 2
        "I really hate artichokes." --> "really" and "hate" --> -2 * 2 = -4
        "I don't like pizza" --> "like" and "don't" --> 1 * -1 = -1

    a problem i can see is with a sentence like "I don't really like pumpkin pie."
        there are three words of interest here: "don't", "really", and "like"
            ("don't" = -1, "really" = 2, "like" = 1)

        the sentiment for this sentence would be 1 * 2 * -1 = -2, which is equivalent to "hate"
            and i'm not too sure if "I don't really like..." means the same thing as "I hate..."
            maybe "I really don't like..." means about the same as "I hate..."

this is a bit more complex than previously thought
    there are cases like "My sister's boyfriend likes to go hiking." and "My sister's boyfriend's brother went to the Bahamas last month."
    which might require some method of rolling through the input and picking out relations between people(entities)

    the speaker is always either going to directly talk about someone(or something) or talk about someone in terms of relations, all of such relations stemming from the speaker themself
        in the example of "My sister's name is Julie." we see that the speaker is talking in terms of relations
            the speaker is the first entity we find, then Julie/the speaker's sister
            the relation that links the two entities(people) is a familial one, and more specifically "sister"

    in the case of "My sister's boyfriend's brother went to the Bahamas last month."
        again we see the speaker speaking in terms of relations
        
        the first entity is always the speaker, the second is the speaker's sister, the third is the speaker's sister's boyfriend, and the fourth and final is the speaker's sister's boyfriend's brother
        the relations between them is [("family", "sister"), ("romance", "boyfriend"), ("family", "brother")]
        cases like these are impossible to pick out using the current setup(the furthest it would get is sister)

    to tackle more complex cases, it might make sense to go through the input picking out relations
        if no relation is found, then it can be assumed that the speaker is directly talking about someone/something(themself, or a person by name)


for inputs with more complex/chain relations, it might make sense to have an 'indicated' or 'complex' relation label
